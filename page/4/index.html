NIO的缓冲区NIO（New Input/Output）是 Java 中的一种高效 I/O 模型，其核心之一是 缓冲区（Buffer） 的使用。缓冲区通过多种方式提高了 I/O 操作的效率，以下是其工作原理和优势： 1. 数据读写分离在传统的 I/O 中，数据是直接从流中读到目标对象中，或从目标对象写到流中，而 NIO 中的数据读写是基于缓冲区的。 读取数据：从通道（Channel）读取到缓冲区，程序处理缓冲区中的数据。 写入数据：先写入缓冲区，再从缓冲区写入到通道。 这种模式使得数据的管理更加灵活，同时可以减少数据拷贝的次数，从而提升性能。 2. 直接缓冲区（Direct Buffer）优化NIO 提供两种缓冲区类型： 非直接缓冲区：数据存储在 JVM 堆中，读写时需要通过操作系统的拷贝将数据移到本地 I/O 缓冲区。 直接缓冲区：使用操作系统的内存（堆外内存），数据直接在通道和缓冲区之间传输，避免了 JVM 堆与操作系统本地缓冲区之间的拷贝操作。 通过使用...
