1.buildCommentTreeAsync 方法（异步构建评论树）

  • 通过 CompletableFuture.supplyAsync 异步执行 buildTreeStructure 方法，避免阻塞主线程。
  • 先获取 所有顶级评论的 ID（topComments）。
  • 查询 所有子评论（通过 commentDao.listSubCommentsRecursive(articleId, topIds)）。
  • 递归构建评论树，返回 CommentTree 结果。

📌 关键点

  • 异步执行：使用 executor 线程池，提升性能。
  • 数据库查询优化：只查 顶级评论的所有子评论，减少 SQL 查询次数。

2. buildTreeStructure 方法（递归构建评论树）

作用

  • 先将 顶级评论 转换成 TopCommentDTO 并存入 topMap。
  • 遍历所有 子评论，找到其 父评论 并构建层级关系。
  • 使用 Queue 进行层级遍历，支持 无限层级的评论树。
  • 处理无法立即找到父级的子评论（延迟处理）。

📌 关键点

  1. 转换 CommentDO 为 DTO
    • convertTopComments(topComments) → 顶级评论 TopCommentDTO
    • convertSubComment(comment) → 子评论 SubCommentDTO
  2. 双队列处理评论层级
    • Queue processingQueue 存放子评论，逐层处理。
    • Set processedIds 记录已处理的评论，防止重复处理。
  3. 关联评论
    • 如果 **parentId == null**，说明是 顶级评论的子评论，直接加入 TopCommentDTO。
    • 否则找到 parentId 所对应的 SubCommentDTO，加入其 子评论列表。
  4. 延迟处理子评论
    • 如果父评论未找到，说明父评论可能还未处理完成，将子评论重新加入 Queue，稍后处理。
  5. 排序
    • 按创建时间降序排列，让最新的顶级评论 排在前面。

示例

数据库数据

id content topCommentId parentCommentId
1 顶级评论A 1 null
2 顶级评论B 2 null
3 子评论1 1 1
4 子评论2 1 3
5 子评论3 2 2
